# Complete Neovim Shortcuts Guide

This guide covers all custom shortcuts, keybindings, and AI completion features configured in your Neovim setup.

## Quick Reference Commands

- **Show shortcuts cheatsheet**: `<leader>?`
- **Frontend Dev Helper Menu**: `<leader>h`
- **Find keymaps**: `<leader>fk`
- **Find commands**: `<leader>fc`

## AI Completion

### GitHub Copilot (Enabled by Default)

#### Basic Controls
- **Accept suggestion**: `Ctrl+j`
- **Accept word**: `Ctrl+l`
- **Next suggestion**: `Ctrl+k`
- **Previous suggestion**: `Ctrl+h`
- **Dismiss suggestion**: `Ctrl+x`

#### Management Commands
- **Toggle Copilot**: `<leader>cp`
- **Copilot status**: `<leader>cs`
- **Copilot panel**: `<leader>cP`
- **Authenticate**: `:Copilot auth`

#### Copilot Chat
- **Toggle Chat**: `<leader>cc` or `Ctrl+Space`
- **Open Chat with selection**: `Ctrl+Space` (in visual mode)
- **Reset Chat**: `<leader>cr`
- **Explain code**: `<leader>ce` (visual mode)
- **Fix code**: `<leader>cf` (visual mode)
- **Optimize code**: `<leader>co` (visual mode)
- **Write tests**: `<leader>ct` (visual mode)
- **Document code**: `<leader>cd` (visual mode)
- **Quick Chat**: `<leader>cq`

### Codeium (Disabled by Default)

To switch to Codeium:
- **Enable Codeium**: `:UseCodeium` (automatically disables Copilot)
- **Enable Copilot**: `:UseCopilot` (automatically disables Codeium)

When Codeium is enabled:
- **Accept suggestion**: `Tab` or `Ctrl+y`
- **Next suggestion**: `Ctrl+n`
- **Previous suggestion**: `Ctrl+p`
- **Toggle Codeium**: `<leader>ai`

## Basic Text Editing

### Clipboard Operations
- **Select all**: `Ctrl+a`
- **Copy**: `Ctrl+c` (visual mode copies selection, normal mode copies line)
- **Paste**: `Ctrl+v`
- **Undo**: `Ctrl+z` (if not working, try `u`)
- **Redo**: `Ctrl+Shift+z` (if not working, try `Ctrl+r`)

### Text Selection
- **Select left**: `Shift+Left`

- **Select up**: `Shift+Up`

## Leader Key Prefixes

Understanding the leader key organization:
- `<leader>a`: AI/Auto actions
- `<leader>b`: Buffer operations
- `<leader>c`: Code actions
- `<leader>f`: File/Find operations
- `<leader>g`: Git operations
- `<leader>h`: Help/Hydra menus
- `<leader>l`: Live server
- `<leader>n`: npm commands
- `<leader>p`: pnpm commands
- `<leader>q`: Quit/Session
- `<leader>r`: REST/Run operations
- `<leader>s`: Search
- `<leader>t`: Terminal/Tailwind
- `<leader>u`: UI
- `<leader>w`: Windows
- `<leader>x`: Diagnostics/Quickfix
- `<leader>y`: Yarn commands

## Tips

1. **Leader key**: The leader key is typically `Space` or `\` depending on your configuration
2. **Which-key**: Press any prefix key and wait to see available completions
3. **Frontend Dev Helper**: Press `<leader>h` for an interactive menu of common frontend tasks
4. **AI Assistant**: Copilot is enabled by default. Use `:UseCodeium` to switch to the free alternative
5. **Conflicts**: Some keybindings may conflict (e.g., `Ctrl+x` for Copilot dismiss vs delete selection)

## Customization

All keybindings are defined in:
- `~/.config/nvim/lua/keymaps/consolidated.lua` - Basic IDE shortcuts
- `~/.config/nvim/lua/config/keymaps.lua` - Additional custom mappings
- `~/.config/nvim/lua/plugins/` - Plugin-specific keybindings

To add or modify shortcuts, edit these files and reload Neovim.
